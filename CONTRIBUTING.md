# Contributing to Shortify-URL

Thanks for your interest in contributing! A few simple guidelines:

- Fork the repository and create a branch for your change: `git checkout -b feature/my-feature`.
- Keep changes small and focused; include tests where applicable.
- Follow the existing code style and run `mvn test` before opening a PR.
- Open a pull request describing the change; link any related issues.
- Be responsive to review feedback and update your PR as requested.

If you're unsure where to start, check the Issues list and look for `good first issue` tags.
